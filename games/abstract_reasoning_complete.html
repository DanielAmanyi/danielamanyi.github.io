Mission

Identify the transformation rule, not the answer.

Every abstract reasoning question is a search for one or more logical rules. Unlike a personality assessment, this section has objectively correct answers — you're training a skill, not expressing a preference.

The Prime Directive

Do not look for the correct answer. Look for the rule that generated the pattern.

Candidates who search for familiar shapes perform worse than those who search for transformations.

The 12 Rules

1Separate Constants from Variables

Every puzzle contains things that change and things that stay the same. Name the constant first, then the rule only has to explain the variable. See it in the gallery →

2Only One or Two Rules Usually Exist

Don't invent complicated explanations. Most matrices are built from rotation, reflection, counting, alternation, addition/subtraction, overlay, position, fill, or size. If you think you've found five rules, you're probably wrong.

3Examine Rows Before Columns

Most matrices are generated row-wise. If rows don't explain the pattern, check columns. Only then consider diagonals.

4Look for Change, Not Objects

Instead of "there is a triangle," say "the triangle rotated 90°" or "the triangle became striped." Naming the transformation is what solves the puzzle.

5One Feature at a Time

Separate every visual feature — shape, position, size, colour, fill, rotation, direction, number, border, symmetry. Never analyse everything together.

6Count Everything

Many difficult questions are simply counting: shapes, dots, lines, corners, filled objects, empty objects.

7Rotation Has Predictable Angles

Common rotations are 90°, 180°, 270°, clockwise or counter-clockwise. Don't eyeball it — measure mentally.

8Reflection Is Different from Rotation

Many candidates confuse these. Mirror images are not rotations — an asymmetric shape makes the difference obvious. See it in the gallery →

9Eliminate Impossible Answers First

Instead of hunting for the correct answer, remove obviously incorrect ones. If all figures contain exactly 3 objects, discard every option containing 2 or 4.

10Complexity Is Usually Linear

Don't assume magic. Typical sequences run 1, 2, 3, 4 — not 1, 17, 4, 93. Treat this as a strong prior, not an absolute law.

11When Stuck

Ask "what changed?" — not "what is missing?" The first points you at the rule; the second sends you back to guessing.

12Time Is Part of the Test

If you cannot identify the rule within about 45–60 seconds: guess, move, return later if time permits. One difficult question should not cost three easy ones.

Common Mistakes

❌ Looking at the options first. → ✔ Solve first.
❌ Looking at the whole picture. → ✔ Separate attributes.
❌ Guessing the designer's intention. → ✔ Find the mathematical rule.
❌ Overcomplicating. → ✔ Prefer the simplest rule that explains all figures.

Before Every Question

  1. Count objects.
  2. Check rotation.
  3. Check reflection.
  4. Check fill.
  5. Check position.
  6. Check size.
  7. Check alternation.
  8. Check overlay.
  9. Check rows.
  10. Check columns.

This becomes automatic with practice — that's what the mock session below is for.

The Engineer's Mindset

You're not solving a puzzle. You're reverse-engineering an algorithm. Every matrix is effectively a tiny program:

Input
↓
Transformation Rule
↓
Output

Think of each puzzle like debugging a system: the first image is the input, the second shows the state transition, the third should follow the same deterministic rule.

A workplace abstract reasoning question is not testing whether I recognize a shape. It is testing whether I can find the deterministic rule that generated the pattern, under time pressure, without inventing complexity that isn't there.

I will separate constants from variables, examine rows before columns, isolate one feature at a time, and eliminate rather than guess. If I cannot find the rule within a minute, I move on.
